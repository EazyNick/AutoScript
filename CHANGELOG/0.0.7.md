## [0.0.7] - 2026-01-07

### 설정 기능 개선

#### 스크립트 실행 간격 설정
- 기본 타임아웃 설정을 스크립트 실행 간격으로 변경
- 각 스크립트 실행 사이에 설정된 간격만큼 대기

- 기본값: 0.5초
- 전체 실행 시 각 스크립트 사이에 간격 적용

#### 설정 모듈 구조 개선
- `server/settings/` 폴더 생성 및 설정 관련 모듈 통합
  - `execution_interval.py`: 스크립트 실행 간격 관리
  - `theme.py`: 테마 설정 관리
  - `language.py`: 언어 설정 관리
- 설정 관련 함수들을 중앙 집중식으로 관리

### 설정 제거

- 실패 시 자동 재시도 설정 제거
- UI, 서버, DB에서 완전히 제거

#### 알림 설정 제거
- 완료 알림 설정 제거
- 오류 알림 설정 제거

### 주요 변경사항

#### DB 설정 키 변경
- `execution.defaultTimeout` → `execution.scriptInterval`
- 기존 타임아웃 개념에서 실행 간격 개념으로 변경

#### 실행 로직 개선
- 전체 실행 시 각 스크립트 사이에 설정된 간격만큼 대기
- 서버에서 설정값을 조회하여 적용
- 설정이 없을 경우 기본값 0.5초 사용

### 기술적 변경사항

- `server/settings/execution_interval.py`: 스크립트 실행 간격 조회 함수 추가

- `server/api/script_router.py`: 노드 실행 간 간격 적용
- `UI/src/js/components/sidebar/sidebar-scripts.js`: 전체 실행 시 스크립트 간 간격 적용
- `UI/src/pages/workflow/settings.js`: 재시도 및 알림 설정 UI 제거
- `server/main.py`: 재시도 및 알림 설정 초기화 로직 제거

### 엑셀 비교 노드 개선

#### 엑셀 비교 노드 종합 노드화
- 엑셀 비교 노드를 종합 노드로 변경
- 파라미터로 원본/대상 엑셀 파일 경로를 직접 받아서 처리
- 내부에서 엑셀 열기, 시트 선택, 비교, 닫기를 모두 수행
- 별도의 엑셀 열기/닫기 노드 불필요

#### 파라미터 변경
- `source_execution_id`, `target_execution_id` 제거
- `source_file_path`, `target_file_path` 추가 (필수)
- `source_sheet_name`, `target_sheet_name` 추가 (선택, 기본값: "Sheet1")
- `visible`, `save_changes` 파라미터 추가

#### UI 개선
- 엑셀 비교 노드 파라미터에 파일 선택 버튼 추가
- PC 로컬 디렉토리에서 엑셀 파일 선택 가능
- "엑셀 열기 노드 필요" 경고 메시지 제거

## [0.0.7] - 2026-01-08 (260108)

### 노드 패키징 시스템 추가

#### 노드 패키징 기능
- 커스텀 노드를 ZIP 파일로 내보내기 기능 추가
- `scripts/package/package-node.py`: 노드 패키징 스크립트
- `scripts/package-node.bat`: 패키징 배치 파일
- `exports/` 폴더에 `.asnode.zip` 형식으로 저장

#### 노드 설치 기능
- ZIP 파일에서 노드를 자동으로 설치하는 기능 추가
- `shared-nodes/` 폴더의 모든 ZIP 파일 자동 설치 지원
- `scripts/package/install-node.py`: 노드 설치 스크립트
- `scripts/install-node.bat`: 설치 배치 파일
- `--no-skip-duplicate` 옵션으로 기존 노드 덮어쓰기 지원

#### 노드 생성/삭제 도구 개선
- `scripts/nodes/create-node.py`: 노드 생성 스크립트 (기존)
- `scripts/nodes/delete-node.py`: 노드 삭제 스크립트 (기존)
- `scripts/create-test-node.bat`: 테스트 노드 생성 배치 파일
- `scripts/delete-node.bat`: 노드 삭제 배치 파일

### 통합 및 개선

#### 스크립트 통일
- `create-node.py`, `package-node.py`, `install-node.py` 세 스크립트의 동작 방식 통일
- 카테고리별 디렉토리 구조 일관성 유지 (`server/nodes/{category}/`)
- `nodes_config.py` 직접 추가 방식 통일
- 파일명 규칙 통일 (Python: 스네이크 케이스, JavaScript: 케밥 케이스)

#### 인코딩 문제 해결
- Windows 콘솔에서 이모지 출력 시 인코딩 오류 수정
- 모든 배치 파일 및 Python 스크립트의 출력 메시지를 텍스트로 변경
- `[OK]`, `[ERROR]`, `[WARN]`, `[SKIP]` 형식으로 통일

#### 파일 구조
- `exports/`: 패키징된 노드 저장 폴더 (자동 생성)
- `shared-nodes/`: 설치할 노드 ZIP 파일 저장 폴더 (자동 생성)
