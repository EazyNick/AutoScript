# AutoScript 0.1 릴리즈

**릴리즈 날짜**: 2026-01-09

AutoScript 0.1은 시각적 워크플로우 편집기를 통해 노드 기반 자동화 스크립트를 생성하고 실행할 수 있는 자동화 플랫폼의 첫 번째 정식 릴리즈입니다.

## 주요 하이라이트

- **시각적 워크플로우 편집기**: 드래그 앤 드롭으로 노드를 연결하여 복잡한 자동화 작업을 쉽게 구성
- **다양한 노드 타입**: 시작, 액션, 조건, 반복, 대기, 이미지, 엑셀 등 다양한 노드 제공
- **스크립트 관리 시스템**: 대시보드를 통한 스크립트 관리 및 실행 통계 확인
- **패키징 및 공유**: 스크립트와 노드를 ZIP 파일로 패키징하여 공유 가능
- **개발자 도구**: 노드 생성, 삭제, 검증 도구 제공

## 시스템 요구사항

- **Python**: 3.10.6 이상
- **운영체제**: Windows 10/11
- **메모리**: 최소 4GB RAM 권장
- **디스크**: 최소 1GB 권장

## 설치 방법

### 1. 저장소 클론

```bash
git clone <repository-url>
cd AutoScript
```

### 2. 가상환경 생성 및 활성화

```bash
python -m venv venv
venv\Scripts\activate
```

> **중요**: 가상환경 이름은 반드시 `venv`로 생성해야 합니다.

### 3. 패키지 설치

```bash
cd server
pip install -r requirements.txt
```

### 4. 서버 실행

프로젝트 루트에서 `start-server.bat` 파일을 실행하거나:

```bash
start-server.bat
```

서버가 실행되면 브라우저에서 `http://127.0.0.1:8001`로 접속하여 웹 UI를 사용할 수 있습니다.

**자세한 설치 가이드**: [설치 및 실행 가이드](../INSTALLATION.md)

## 빠른 시작

### 첫 번째 워크플로우 만들기

1. 웹 UI에서 "노드 추가" 버튼을 클릭하거나 `Ctrl+N` 키를 누릅니다
2. 원하는 노드 타입을 선택합니다 (시작, 클릭, 대기 등)
3. 노드를 캔버스에 배치하고 연결합니다
4. 각 노드의 설정을 조정합니다 (노드 더블클릭 또는 설정 버튼 클릭)
5. `Ctrl+S`로 워크플로우를 저장합니다
6. `F5` 또는 `Ctrl+R`로 워크플로우를 실행합니다

**자세한 사용 가이드**: [사용자 가이드](../docs/USER_GUIDE.md)

## 주요 기능

### 시각적 워크플로우 편집기

드래그 앤 드롭으로 노드를 배치하고 연결하여 자동화 스크립트를 만들 수 있습니다. 노드 간 데이터 전달과 실행 순서를 시각적으로 확인할 수 있으며, 노드 크기 변경 시 자동으로 충돌을 감지하고 방지합니다.

**관련 문서:**
- [사용자 가이드](../docs/USER_GUIDE.md)
- [워크플로우 구조](../docs/dev/workflow-structure.md)

### 다양한 노드 타입

시작 노드, 액션 노드, 조건 노드, 반복 노드, 대기 노드, 이미지 노드, 엑셀 노드 등 다양한 타입의 노드를 제공합니다. 엑셀 비교 노드는 종합 노드로 구현되어 단일 노드로 엑셀 파일 열기, 시트 선택, 비교, 닫기 작업을 모두 수행할 수 있습니다.

**관련 문서:**
- [노드 목록](../docs/node.md)
- [노드 시스템 문서](../docs/nodes/README.md)

### 스크립트 관리 시스템

생성한 워크플로우를 스크립트로 저장하고 관리할 수 있습니다. 대시보드를 통해 전체 스크립트 목록, 실행 횟수, 실패 횟수, 활성화 상태 등을 확인할 수 있으며, 스크립트별로 활성화/비활성화를 설정하여 전체 실행 시 제어할 수 있습니다.

**관련 문서:**
- [대시보드](../docs/dev/ui-ux/dashboard.md)

### 실행 모니터링

스크립트 실행 중 실시간으로 진행 상황을 모니터링할 수 있습니다. 실행 중인 노드를 시각적으로 강조 표시하며, 실행 결과와 로그를 확인할 수 있습니다.

**관련 문서:**
- [실행 로그 흐름](../docs/dev/execution-log-flow.md)
- [노드 실행 흐름](../docs/dev/node-execution-flow.md)

### 패키징 및 공유 시스템

스크립트와 커스텀 노드를 ZIP 파일로 패키징하여 다른 환경으로 이동하거나 다른 사용자와 공유할 수 있습니다. 스크립트 패키징 시 필요한 커스텀 노드가 자동으로 감지되어 함께 포함됩니다.

**관련 문서:**
- [스크립트 패키징 가이드](../docs/dev/script-packaging-guide.md)

### 설정 관리 시스템

스크립트 실행 간격, 테마, 언어 등의 설정을 중앙 집중식으로 관리합니다. 데이터베이스에 저장되어 영구적으로 유지됩니다.

노드 생성 스크립트를 실행하면 Python 노드 클래스, JavaScript 렌더링 파일, 설정 파일이 자동으로 생성되며, 필요한 설정도 자동으로 추가됩니다. 생성된 템플릿에는 기능 구현 구역이 명확히 표시되어 있어 개발자가 쉽게 기능을 구현할 수 있습니다.

## 사용자 경험

### 직관적인 인터페이스

드래그 앤 드롭으로 쉽게 워크플로우를 구성할 수 있으며, 노드 설정 모달을 통해 파라미터를 쉽게 입력할 수 있습니다. 파일 선택 버튼을 통해 로컬 디렉토리에서 파일을 직접 선택할 수 있으며, 이전 노드의 출력 값을 변수로 선택할 수 있는 기능도 제공합니다.

### 키보드 단축키

- `Ctrl+N`: 노드 추가
- `Ctrl+S`: 워크플로우 저장
- `Ctrl+Z`: 노드 이동 되돌리기
- `F5` / `Ctrl+R`: 워크플로우 실행
- `Delete`: 선택한 노드 삭제
- `Esc`: 실행 중지

### Undo/Redo 기능

노드 이동과 삭제에 대한 Undo 기능을 제공합니다. 최대 20개의 작업 기록까지 되돌릴 수 있으며, 삭제된 노드의 연결 정보까지 완전히 복원됩니다.

**관련 문서:**
- [Undo/Redo 시스템](../docs/dev/undo-redo-system.md)

경계 노드(시작/종료)만 먼저 로드하고 나머지 노드는 지연 로드하여 초기 로딩 속도를 개선했습니다. 중복 코드를 제거하고 공통 함수로 통합하여 코드 재사용성을 향상시켰습니다.
