# 스크립트 로드 시스템 문서

이 폴더는 워크플로우 편집기에서 스크립트(워크플로우)를 로드하는 시스템에 대한 문서를 포함합니다.

## 문서 목록

### [스크립트 로드 워크플로우](./script-loading-workflow.md)

스크립트 로드의 전체 프로세스를 설명하는 메인 문서입니다.

**주요 내용**:
- 아키텍처 및 컴포넌트 구조
- 로드 워크플로우 상세 설명
- 취소 메커니즘 (Race Condition 해결)
- 노드 제거 프로세스
- 에러 처리 및 성능 최적화

## 주요 기능

### 1. 빠른 스크립트 전환 지원

빠르게 여러 스크립트를 전환해도 노드가 섞이지 않도록 보장합니다.

**핵심 메커니즘**:
- 고유 로드 ID로 각 로드 추적
- 이전 로드 완전 취소 및 노드 제거
- 노드 제거 완료 후에만 새 노드 추가

### 2. 노드 레지스트리 로딩 보장

스크립트 로드 전에 노드 레지스트리가 완전히 로드되도록 보장합니다.

**확인 지점**:
- `WorkflowPage.onScriptChanged()`
- `PageRouter.initEditor()`
- `AddNodeModal.show()`

### 3. 안정적인 노드 제거

기존 노드를 완전히 제거한 후 새 노드를 추가합니다.

**제거 순서**:
1. 연결선 제거
2. NodeManager에서 제거
3. DOM에서 제거
4. 데이터 초기화
5. DOM 업데이트 완료 대기

## 관련 컴포넌트

- **WorkflowLoadService**: 로드 서비스 메인 로직
- **WorkflowPage**: 워크플로우 페이지 컨트롤러
- **PageRouter**: 페이지 라우터 및 초기화
- **NodeRegistry**: 노드 타입 정의 관리

## 개선 이력

### 최근 개선 사항

1. **Race Condition 해결** (2026-01-17)
   - 빠른 스크립트 전환 시 노드가 섞이는 문제 해결
   - 로드 취소 메커니즘 강화

2. **노드 레지스트리 로딩 보장** (2026-01-17)
   - 스크립트 로드 전 노드 레지스트리 로딩 완료 대기
   - 팔레트에 노드가 표시되지 않는 문제 해결

3. **노드 제거 완료 보장** (2026-01-17)
   - 노드 제거를 비동기 함수로 변경
   - DOM 업데이트 완료 보장
